Codio Activity : Buffer Overflow

In this activity, there are 2 set of programs written in C and Python that explains the concept of Buffer Overflow. In the first part the following program is given:

                        #include <stdio.h> 
int main(int argc, char **argv)
{
    char buf[8]; // buffer for eight characters
    printf("enter name:"); 
    gets(buf); // read from stdio (sensitive function!)
    printf("%s\n", buf); // print out data stored in buf
    return 0; // 0 as return value
}

When we run this program on our system, and give the input of length not more than 8, we do not get any error:

codio@spiritmission-botaniccello:~/workspace$ ./bufoverflow
Enter name: Anoushka
Anoushka

But when we run the program and the input is more than 8 characters, we get the error of stack smashing detected:

codio@spiritmission-botaniccello:~/workspace$ ./bufoverflow<
Enter name: Anoushka Pal
Anoushka Pal
*** stack smashing detected ***: <unknown> terminated
Aborted (core dumped)

This shows that the C program (GCC compiler) was able to detect the buffer overflow and it terminated the program. However, it was seen that the input of length 12 was recieved and printed before the error was displayed.

In Part-2, the following Python program is given:

buffer=[None]*10
for i in range (0,11):
    buffer[i]=7
print(buffer)


 when we run the program, we get the following output:

codio@spiritmission-botaniccello:~/workspace$ python3 Overflow.py
Traceback (most recent call last):
File "overflow.py", line 3, in <module>
buffer[i]=7
IndexError: list assignment index out of range

It also shows that the Python interpreter was able to detect the buffer overflow and stopped after 8 iterations, the for loop stopped.
